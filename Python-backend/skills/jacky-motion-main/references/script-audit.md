# 口播稿检查

判断口播稿能否被动画化。稿子弱，HTML 再漂亮也没用。

## 通过标准（6条）

1. **小白能听懂**：非专业观众能跟上术语、因果链和结论。
2. **每段一个重点**：每个自然段只承载一个核心信息。
3. **有可视化抓手**：有对比、顺序、因果、数字、层级、时间线、关系、引用、流程、前后变化。
4. **信息密度够**：每个节拍都说了具体内容，不是水话或空转过渡。
5. **口语化自然**：像真人讲解，不像文章朗读或AI模板句。
6. **能匹配视觉动词**：每个节拍都能分配高亮/连接/增长/对比/推近/展开/流动/堆叠/计数/灰化聚焦中的一个。

## 常见问题

| 问题 | 表现 | 修法 |
|---|---|---|
| 太抽象 | "本质上是效率提升"，没有例子 | 加具体例子、数字、流程或对比 |
| 一段太多信息 | 一个段落同时讲历史、问题、方案、结论 | 拆成多个节拍 |
| 文字太满 | 屏幕需要展示整段文字 | 改成一个口播重点+一个屏幕焦点 |
| 没有视觉动作 | 只是承上启下的过渡句 | 合并到前后节拍或删掉 |
| 术语太多 | 先堆概念再解释 | 先给白话定义或类比 |
| AI腔明显 | "接下来我们将深入探讨……" | 改成直接自然的人话 |

## 改写规则

1. 保留用户原本的观点、事实和论证方向。
2. 改成适合说出来的中文，不改成营销文案。
3. 保留关键数字、名称、日期、来源说法。
4. 稠密段落可以拆成多个节拍。
5. 只在用户内容已暗示的范围内添加视觉锚点，不编造事实。
