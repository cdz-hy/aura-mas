# 验收检查

交付前逐条检查，不通过则自动修复。

1. 口播稿是否经过审查
2. 每个节拍是否有明确的视觉动词
3. 每个节拍是否只有一个核心信息
4. 屏幕文字是否短于口播
5. 是否全篇单一风格
6. 每个节拍是否撑满画面（标题 ≥ 96px / 4:3 ≥ 72px，正文 ≥ 25px，无大面积留白）
7. 动画是否服务于信息理解（非装饰）
8. 每个节拍是否有至少 2 个动画阶段（非整页出现）
9. 是否存在捏造的数据/来源/引用
10. 首屏是否建立了风格锚点
11. 文字录屏后是否可读（标题 ≥ 96px / 4:3 ≥ 72px，正文 ≥ 25px，标签 ≥ 18px）
12. 手动模式下节拍动画结束后是否停住
13. body 背景色是否与 stage 主背景色相同（缩放后不露底色）
14. 导航栏是否在 #stage 内部用 position:absolute（非 fixed）
15. 所有内容是否在 stage 边界内（无溢出裁切）
16. 每个 beat 是否有视觉重心（图文 beat：截图 ≥ 40% 面积或标题 ≥ 正文 3 倍；纯文字 beat：主标题 ≥ 120px / 4:3 ≥ 88px）——禁止所有元素同等大小平铺
17. 信息截图是否完整显示（无 object-fit:cover 裁切），容器是否无多余 background-color（无灰色背景框）
