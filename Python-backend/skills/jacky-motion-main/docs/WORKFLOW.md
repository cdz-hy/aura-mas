# 6 阶段工作流

```text
口播稿 → 审稿 → 分镜 → 锁风格 → 生成 HTML → 验收 → 配音录制
```

## Phase 1：审稿

检查脚本是否适合直接做动画。输出通过、轻改或重写建议。

重点看：

- 主题是否单一。
- 信息密度是否过高。
- 逻辑跳跃是否明显。
- 是否存在无法视觉化的抽象段落。

## Phase 2：分镜

按口播自然段落拆节拍。每个节拍必须包含：

- 口播摘要
- 一句核心信息
- 信息结构
- 视觉动词
- 屏幕文字

## Phase 3：锁风格

根据内容类型推荐一种风格，并为每个 beat 描述构图方案。整片只用一种风格。

## Phase 4：生成 HTML

读取对应风格模板，生成单文件 16:9 HTML。HTML 默认支持点击推进和键盘控制。

## Phase 5：验收

检查：

- 文字是否溢出。
- 截图是否完整显示。
- 页面是否适合录屏。
- 动画是否服务口播节奏。
- 是否停在可读的 hero frame。

## Phase 6：配音录制

两种方式：

- 自行配音：浏览器全屏打开 HTML，用 QuickTime、OBS 或系统录屏。
- TTS 配音：生成音频文件，注入 HTML，启用自动播放模式后录屏。
